### Usage Examples

#### Upload English Subtitles
```bash
curl -X POST http://localhost:7001/api/subtitles/upload \
  -F "subtitle=@english.vtt" \
  -F "lessonId={lessonGuid}" \
  -F "userId={instructorGuid}" \
  -F "language=en" \
  -F "label=English" \
  -F "isDefault=true"
```

#### Upload Simplified Chinese Subtitles
```bash
curl -X POST http://localhost:7001/api/subtitles/upload \
  -F "subtitle=@chinese-simplified.vtt" \
  -F "lessonId={lessonGuid}" \
  -F "userId={instructorGuid}" \
  -F "language=zh-Hans" \
  -F "label=简体中文" \
  -F "isDefault=false"
```

#### Upload Arabic Subtitles
```bash
curl -X POST http://localhost:7001/api/subtitles/upload \
  -F "subtitle=@arabic.vtt" \
  -F "lessonId={lessonGuid}" \
  -F "userId={instructorGuid}" \
  -F "language=ar" \
  -F "label=العربية" \
  -F "isDefault=false"
```

### Best Practices

1. **Use Standard Codes**: Always use the 2-letter ISO 639-1 code (e.g., "en" not "eng")
2. **Provide Native Labels**: Use the native script for labels (e.g., "日本語" not "Nihongo")
3. **Set One Default**: Only one subtitle should be marked as default per lesson
4. **Regional Variants**: Use extended codes for regional variants (e.g., "zh-Hans", "pt-BR", "es-MX")
5. **Right-to-Left Languages**: Arabic, Hebrew, Urdu, Farsi require RTL text direction in WebVTT

### WebVTT Example for RTL Languages

```vtt
WEBVTT

NOTE Right-to-left subtitle example (Arabic)

00:00:00.000 --> 00:00:05.000 align:start
مرحبا بكم في درس إنسايت ليرن

00:00:05.000 --> 00:00:10.000 align:start
اليوم سنتعلم عن الترجمة متعددة اللغات
```

### Language Detection Tools

For automatic language detection in uploaded files:
- **Library**: `Lingua` (NuGet package)
- **API**: Google Cloud Translation API
- **Parser**: Extract language from WebVTT `Language:` header
